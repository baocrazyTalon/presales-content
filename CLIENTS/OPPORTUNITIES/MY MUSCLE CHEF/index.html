01 — Architecture

MACH Architecture & API-First Approach

Talon.One is built as a MACH-compliant promotion engine — Microservices, API-first, Cloud-native, and Headless. Talon.One is hosted on Google Cloud Platform (GCP), while My Muscle Chef's in-house headless ecommerce stack runs on AWS — both connected seamlessly via API.

🔲

Microservices

Independent, loosely-coupled services that scale individually. Each domain (rules, sessions, profiles, loyalty) operates autonomously.

⚡

API-First

Every feature is exposed via a REST API. No UI dependency — your backend calls Talon.One directly. Full control, full flexibility.

☁️

Cloud-Native

SaaS platform deployed on Google Cloud Platform (GCP). Auto-scaling, high availability, managed infrastructure. GCP Australia data center for low latency.

🔗

Headless

No frontend imposed. Talon.One is a pure backend engine — your website, app, or POS calls the API and renders the experience your way.

Why this matters for My Muscle Chef: Your team already builds and maintains a custom headless ecommerce platform. Talon.One slots in as a backend API service — no iframes, no widgets, no frontend lock-in. Your developers call the Integration API from your existing services and control every aspect of the user experience.

02 — Environments

Sandboxes & Production Applications

Talon.One provides isolated Applications that map to your development lifecycle. Each Application has its own campaigns, rules, API keys, and data — completely independent.

🛠 Development Sandbox for dev team exploration & unit testing
🧪 QA / Test Dedicated app for QA validation & automated tests
🚀 Staging Pre-production mirror for final integration checks
🔴 Production Live environment serving real customers

Key Benefits

  • Each application has separate API keys — no risk of dev changes hitting production
  • Campaign rules can be built and tested in sandbox before promoting to production
  • Sandbox applications are included in your Talon.One license at no extra cost
  • Each environment can be mapped to your existing AWS dev / staging / prod infrastructure
  • Data is fully isolated between applications
03 — Unique Identifiers

IntegrationID — Your System of Record

Every entity in Talon.One (customer profiles, sessions, coupons, referrals) is identified by an IntegrationID — a unique string provided by you, not generated by Talon.One.

👤

Customer Profile IntegrationID

Use your existing customer ID from your database. Example: MMC-CUST-98421

This becomes the key to link loyalty points, tiers, and cashback to the right person.

🛒

Session IntegrationID

Use your order/cart ID. Example: ORDER-2026-03-8842

Ensures idempotency — same session ID always returns the same result.

🏷️

Coupon & Referral Codes

Codes generated by Talon.One or imported from your system. Tied to campaigns and validated in real-time.

Why this matters: My Muscle Chef retains full ownership of identifiers. No vendor lock-in on IDs. Your customer ID in your database is the same ID used in Talon.One — making reconciliation, debugging, and data exports straightforward.

04 — Core Integration

Session Integration API

The Session API is the heart of Talon.One integration. It represents a shopping session / cart / order and evaluates all active campaign rules in real-time.

Endpoint

PUT /v2/customer_sessions/{integrationId}

Session States

🟢 Open Cart is active, customer is shopping
→
🔵 Closed Order confirmed / paid
or →
🔴 Cancelled Order cancelled / refunded
StateWhen to useWhat happens in Talon.OneLoyalty Impact (Cashback)
open Cart updated, item added/removed, coupon applied Campaigns evaluated, effects returned (discounts, cashback preview, tier check) Cashback is pending — shown to customer but not yet credited
closed Order placed & payment confirmed Session locked, effects finalized, loyalty balance updated Cashback is credited to customer's loyalty balance. Spend counted toward tier progression.
cancelled Order cancelled or fully refunded All effects rolled back automatically Cashback is reversed. Spend removed from tier calculation.

Example — Open Session (Cart Update)

// PUT /v2/customer_sessions/ORDER-2026-03-8842 { "customerSession": { "profileId": "MMC-CUST-98421", "state": "open", "cartItems": [ { "name": "Grilled Chicken & Brown Rice", "sku": "MEAL-GCR-001", "quantity": 3, "price": 12.95, "category": "Ready Meals" }, { "name": "Protein Pancakes", "sku": "MEAL-PP-042", "quantity": 2, "price": 9.95, "category": "Breakfast" } ], "couponCodes": ["WELCOME10"], "total": 58.75 } }

Response — Effects for MMC to Render

// Talon.One returns effects to apply { "effects": [ { "effectType": "setDiscount", "props": { "name": "Coupon Discount", "value": 5.87 } }, { "effectType": "addLoyaltyPoints", "props": { "name": "Cashback Earned", "programId": 12, "value": 2.64, "subLedgerId": "main" } } ], "customerProfile": { "loyaltyMemberships": [ { "programId": 12, "currentTier": "Silver", "points": 142.50 } ] } }

Cashback Loyalty Flow: When a session is open, Talon.One returns a cashback preview (e.g., "$2.64 cashback on this order"). When the session is closed, the cashback is credited to the customer's loyalty ledger. If the session is cancelled, the cashback is automatically reversed.

05 — Custom Events

Event API — Non-Transactional Triggers

Use the Event API to send non-cart events into Talon.One. These can trigger campaigns, award loyalty points, or track engagement — without a session/order context.

Endpoint

POST /v2/events

Use Cases for My Muscle Chef

📝

Account Registration

Award welcome cashback when a new customer signs up. Event: account_created

⭐

Product Review

Reward customers for leaving meal reviews. Event: review_submitted

📱

App Download

Bonus cashback for downloading the MMC app. Event: app_installed

🎂

Birthday Milestone

Trigger birthday cashback bonus. Event: birthday_reached

Example Request

// POST /v2/events { "profileId": "MMC-CUST-98421", "type": "review_submitted", "attributes": { "product_sku": "MEAL-GCR-001", "rating": 5, "channel": "website" } }
06 — Customer Data

Customer Profile API

Update customer attributes in Talon.One to power segmentation and targeting within campaign rules. Profiles can be updated via the Session API (automatically) or the dedicated Profile endpoint.

Endpoint

PUT /v2/customer_profiles/{integrationId}

Example — Updating Customer Attributes

// PUT /v2/customer_profiles/MMC-CUST-98421 { "attributes": { "dietary_preference": "high_protein", "subscription_plan": "weekly_10", "city": "Sydney", "state": "NSW", "lifetime_orders": 47, "last_order_date": "2026-03-20T10:30:00Z", "preferred_meals": ["chicken", "fish"] } }

Attributes Power Campaign Rules

Once attributes are synced, the marketing team can create rules such as:

  • "If dietary_preference = high_protein" → 2x cashback on new Protein range
  • "If city = Melbourne" → Free delivery cashback for Melbourne launch
  • "If lifetime_orders > 50" → VIP bonus cashback tier
  • "If subscription_plan = weekly_10" → Subscriber-only exclusive offers
07 — Customer Inventory

Customer Inventory Endpoint

Retrieve the latest customer loyalty data from Talon.One — including cashback balance, active coupons, tier status, and referral data. Use this to render your loyalty dashboard.

Endpoint

GET /v2/customer_profiles/{integrationId}/inventory

What You Get Back

💰

Loyalty / Cashback Balance

Current cashback balance, pending cashback, tier name, tier expiry, and points history per sub-ledger.

🎟️

Active Coupons

All personal coupons allocated to this customer — including status, expiry, and usage count.

📊

Giveaway Items

Free products or gifts awarded through campaigns that the customer hasn't redeemed yet.

🏆

Tier Information

Current tier, tier progress, spend needed for next tier, tier upgrade/downgrade dates.

Example Response (Simplified)

{ "loyalty": { "programId": 12, "programName": "MMC Rewards", "currentTier": "Silver", "ledger": { "currentBalance": 142.50, // Available cashback ($) "pendingBalance": 12.30, // From open sessions "totalEarned": 385.00, // Lifetime earnings "totalSpent": 242.50 // Redeemed cashback } }, "coupons": [ { "code": "SILVER-BIRTHDAY-2026", "expiryDate": "2026-04-15" } ] }

Rendering the loyalty dashboard: MMC front-end calls the inventory endpoint on page load to display the customer's cashback wallet, current tier, and any available coupons. This is fully real-time.

Sync your product catalog to Talon.One so campaign rules can reference product attributes like category, nutritional info, brand, or any custom attribute. It also enables strikethrough pricing on product listing pages.

Sync Methods

MethodBest ForHow
Catalog API (Real-time) Individual product updates PUT /v1/catalogs/{catalogId}/items — upsert products on change
Bulk CSV Upload Initial load or periodic full sync Upload CSV via Management API or Campaign Manager UI
Cart Item Attributes Dynamic attribute at checkout Send item attributes directly inside the session cartItems payload

09 — Customer.io Integration

Integration with Customer.io

Talon.One integrates with Customer.io through webhooks and broadcast API. This creates a powerful loop: Talon.One triggers messaging via Customer.io, and Customer.io pushes audience segments back into Talon.One.

  ┌───────────────────┐         Webhooks          ┌───────────────────┐
  │                   │ ──────────────────────► │                   │
  │                   │  • Trigger campaigns     │                   │
  │    Talon.One      │  • Loyalty data updates  │  Customer.io      │
  │                   │                          │                   │
  │                   │ ◄────────────────────── │                   │
  │                   │     Broadcast API        │                   │
  └───────────────────┘  • Segment sync          └───────────────────┘

1. Webhooks: Talon.One → Customer.io (Event Triggers)

When a campaign rule fires in Talon.One, it can send a webhook to Customer.io to trigger automated messaging workflows.

📩

Campaign Event Trigger

Send events to Customer.io when loyalty milestones are reached. Example: "Customer reached Gold tier → trigger congratulations email + push notification."

💸

Cashback Notification

Notify customers when cashback is credited. Example: "$5.20 cashback earned on your order → send email with balance update."

🎟️

Coupon Delivery

When Talon.One generates a personal coupon, webhook to Customer.io to deliver it via email/SMS/push.

2. Webhooks: Talon.One → Customer.io (Profile Updates)

Keep Customer.io people profiles enriched with real-time loyalty data for targeting.

// Webhook payload from Talon.One to Customer.io // Endpoint: https://track.customer.io/api/v1/customers/{profileId} { "loyalty_tier": "Silver", "cashback_balance": 142.50, "lifetime_cashback_earned": 385.00, "last_purchase_date": "2026-03-20", "next_tier": "Gold", "spend_to_next_tier": 250.00 }

This allows Customer.io to build segments like "Silver members within $50 of Gold" for tier upgrade nudge campaigns.

3. Customer.io → Talon.One (Segment Broadcast)

Customer.io can push audience segments into Talon.One via the Broadcast API. This allows Talon.One campaigns to target customers based on Customer.io segments.

Customer.io Build segment (e.g., "Churning customers")
→
Broadcast API Push segment members to Talon.One
→
Talon.One Campaign Target "Churning" segment with win-back cashback

Powerful Loop: Talon.One runs promotions & loyalty → sends data to Customer.io → Customer.io sends communications & builds segments → segments flow back to Talon.One for targeting. This eliminates data silos between your promotion and messaging platforms.

10 — Segmentation Without Customer.io

API-Based Audience Import

If Customer.io is not available or preferred for segment sync, Talon.One offers direct APIs to import and manage audiences programmatically.

Option A: Update Customer Attributes

Set segment-related attributes directly on customer profiles:

PUT /v2/customer_profiles/MMC-CUST-98421 { "attributes": { "segment_churn_risk": true, "segment_high_value": true, "segment_new_customer": false } }

Campaign rules then use these attributes: "If segment_churn_risk = true → 15% bonus cashback".

Option B: Audiences via Management API

Create and manage audiences in Talon.One and import customer lists:

// Create audience POST /v1/audiences { "name": "High Value Customers", "integration_id": "seg-high-value" } // Add customers to audience POST /v2/audiences/seg-high-value/memberships { "profileIntegrationIds": ["MMC-CUST-98421", "MMC-CUST-12045", ...] }

This allows your data team to run batch jobs (e.g., nightly ETL) that compute segments in your data warehouse and push them into Talon.One.

13 — Security

Security Concepts

Talon.One implements industry-standard security practices to protect your data and API interactions.

🔑

API Key Authentication

Each Application has a unique API key. Integration API uses Api-Key header. Management API uses session tokens or API keys with scoped permissions.

🔒

TLS / HTTPS Only

All API communication is encrypted in transit via TLS 1.2+. No plaintext connections accepted.

🛡️

Role-Based Access Control

Granular roles for Campaign Manager UI — Marketing can manage campaigns, Finance views reports, Admins control configuration.

📋

Audit Logs

Full audit trail of all actions taken in the Campaign Manager and via API — who changed what and when.

🗄️

Data Encryption at Rest

All data stored in Talon.One is encrypted at rest using AES-256. Database backups are also encrypted.

🌏

Data Residency

Talon.One runs on GCP with regional deployments. For MMC, data will be hosted on GCP Australia (Sydney), meeting Australian privacy and data residency requirements.

📜

Compliance

SOC 2 Type II certified. GDPR compliant. Regular penetration testing and vulnerability assessments.

⏱️

Rate Limiting & DDoS Protection

Built-in rate limiting per API key. Infrastructure-level DDoS mitigation. High-throughput plans available for peak events (e.g., MMC sales).
