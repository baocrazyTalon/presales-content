Talon.One Integration with Gcash for a cashback / offer program launch in 2026

This document outlines the proposed technical architecture, integration points, and project plan for implementing Talon.One for a launch of Gcash Loyalty Program in 2026.

Promotions & Loyalty API-First Scalable Infrastructure Composable

01 — Architecture

MACH Architecture & API-First Approach

Talon.One is built as a MACH-compliant promotion engine — Microservices, API-first, Cloud-native, and Headless. It will power both the loyalty program (points, tiers) and all discounts & promotions, consolidating into a single, unified engine for GCash's B2B platform. Talon.One connects seamlessly to GCash's existing systems via API.

🔲

Microservices

Independent, loosely-coupled services that scale individually. Each domain (rules, sessions, profiles, loyalty) operates autonomously.

⚡

API-First

Every feature is exposed via a REST API. No UI dependency — your backend calls Talon.One directly. Full control, full flexibility.

☁️

Cloud-Native

SaaS platform deployed on Google Cloud Platform (GCP). Auto-scaling, high availability, managed infrastructure. APAC region for low latency across GCash's markets.

🔗

Headless

No frontend imposed. Talon.One is a pure backend engine — your B2B portal, mobile app, or ERP calls the API and renders the experience your way.

Why this matters for GCash: Your team manages a complex B2C offer/loyalty program. Talon.One slots in as a backend API service — no iframes, no widgets, no frontend lock-in. Your developers call the Integration API from your existing services and control every aspect of the user experience.

Full Solution to support GCash project

Talon.One is proposing a complete, end-to-end solution — not just a promotion engine. It combines the Talon.One SaaS platform with a purpose-built middleware layer that handles all data flows between GCash's systems, its merchant ecosystem, and the Talon.One rule engine. GCash gets a fully integrated stack: campaign management, loyalty ledgers, and real-time decisioning on one side — seamless connectivity to customer profiles, order management, and merchant data on the other. No integration gaps, no undifferentiated heavy lifting.

03 — Cashback Program

Cashback Program

Talon.One supports GCash's B2C Cashback Program through flexible, configurable loyalty ledgers that can track cashback, tiers, and rewards.

🏆

Cashback Ledger

Each customer profile maintains a real-time cashback balance. Cashback can be awarded on payment, or expired based on configurable rules — all without custom code.

🎖️

Tier Management

Define Bronze, Silver, and Gold tiers with automatic promotion and demotion logic based on rolling spend windows or point thresholds. Tier status unlocks exclusive campaigns and multipliers.

Merchant-Funded Offers

Beyond cashback, GCash's program includes a merchant-funded offer model — the most common pattern in payment app ecosystems. Merchants fund campaigns that reward GCash users with cashback or discounts when they pay with the GCash app at participating merchants. Talon.One manages the full offer lifecycle: eligibility, redemption, and budget control, all configurable without engineering work.

🏪

Merchant-Funded Campaigns

Each participating merchant funds their own promotion budget within Talon.One. Cashback rates, discount amounts, and spending caps are configured per merchant — ensuring GCash never subsidises merchant offers and each campaign is financially ring-fenced.

🎟️

Offer Codes & Vouchers

Merchants can run targeted campaigns using unique or batch-generated offer codes distributed via GCash's app, push notifications, or partner channels. Codes can be single-use, limited-redemption per user, or time-gated — all enforced automatically at the point of payment.

💸

Cashback & Discount at Payment

When a customer pays with GCash at a participating merchant, Talon.One evaluates eligibility in real time and returns the applicable cashback amount or discount. Rules can target by merchant category, transaction amount, time of day, user segment, or payment method.

📊

Budget & Spend Controls

Each merchant campaign has its own coupon budget and spend limit. Talon.One tracks redemptions in real time and automatically stops issuing rewards once the merchant's funding cap is reached — protecting both GCash and merchant from overspend.

04 — Identifiers

Identifiers — Your System of Record

Customer profiles and sessions are identified by an IntegrationID — a unique string provided by GCash, not generated by Talon.One.

👤

Customer Profile Identifier

Use your existing customer ID from GCash. Example: GC-USER-98421

This becomes the key to link cashback, tiers, and rewards to the right individual customer in Talon.One.

🛒

Session Identifier

Use your payment or transaction ID from GCash. Example: PAYMENT-2026-06-8842

Ensures idempotency — the same session ID always returns the same promotion result, preventing double cashback or discount payouts.

Why this matters: GCash retains full ownership of identifiers. No vendor lock-in on IDs. Your customer ID in your CRM or ERP is the same ID used in Talon.One — making reconciliation, debugging, and data exports straightforward.

Data security: Using an opaque, anonymous identifier also adds a layer of data security: Talon.One never stores personally identifiable information (PII) or payment card data (PCI) — keeping sensitive customer data exclusively within GCash's own systems.

05 — Customers Experience and API

Customers Experience and API

05a — Customer Registration

Customer Registration and Update

Update customer attributes in Talon.One to power segmentation and targeting within campaign rules. Profiles can be updated via the Session API (automatically) or the dedicated Profile endpoint.

You can also sync audience memberships in real-time — push your segment/audience IDs alongside profile attributes so Talon.One campaign rules can target customers based on your own segmentation logic (e.g. from your CDP, CRM, or Clevertap).

Endpoint

PUT /v2/customer_profiles/{integrationId}

Example — Updating Customer Attributes

// PUT /v2/customer_profiles/GC-USER-98421 { "attributes": { "account_status": "verified", "kyc_level": "fully_verified", "wallet_tier": "Silver", "preferred_merchant_categories": ["food_and_beverage", "grocery"], "lifetime_spend": 25400, "last_transaction_date": "2026-04-28T14:22:00Z", "payment_frequency": "high" } }

With Audience Sync: Include the audiencesChanges object to add or remove audience memberships in real-time. Talon.One will evaluate campaign rules based on these audience IDs — enabling targeted promotions for specific customer segments from your CDP, CRM, or Clevertap.

// PUT /v2/customer_profiles/GC-USER-98421 { "attributes": { "account_status": "verified", "kyc_level": "fully_verified", "wallet_tier": "Silver", "preferred_merchant_categories": ["food_and_beverage", "grocery"], "lifetime_spend": 25400, "last_transaction_date": "2026-04-28T14:22:00Z" }, "audiencesChanges": { "adds": [ 14 // Add to audience ID 14 (e.g. "Silver Wallet Users") ], "deletes": [ 8 // Remove from audience ID 8 (e.g. "New Users") ] } }

Attributes Power Campaign Rules

Once attributes are synced, the marketing team can create rules such as:

  • "If wallet_tier = Gold" → 2× cashback rate at participating merchants
  • "If kyc_level = fully_verified" → Access to exclusive verified-user offers
  • "If lifetime_spend > 50000" → VIP cashback tier with higher earn rates
  • "If preferred_merchant_categories includes grocery" → Targeted grocery merchant offers

Bulk Update — Multiple Customer Profiles

Update (or create) up to 1,000 customer profiles in a single request using the batch endpoint. Ideal for initial data migration, nightly syncs from your CRM or ERP, or bulk attribute updates.

The integrationId must be a stable identifier. Do not use an ID the customer can update themselves — use a database or ERP ID.

PUT /v2/customer_profiles
// Batch update up to 1,000 profiles { "customerProfiles": [ { "integrationId": "GC-USER-98421", "attributes": { "wallet_tier": "Silver", "kyc_level": "fully_verified" } }, { "integrationId": "GC-USER-10234", "attributes": { "wallet_tier": "Gold", "kyc_level": "fully_verified" } } // ... up to 1,000 profiles per request ] }

05b — Transaction

Transaction

The Session API is the heart of Talon.One integration. It represents a purchase transaction / payment on EZRx and evaluates all active campaign rules in real-time.

Endpoint

PUT /v2/customer_sessions/{customerSessionId}

Session States

🔵 Closed Payment confirmed & settled
or →
🔴 Cancelled Payment cancelled / refunded
StateWhen to useWhat happens in Talon.OneCashback Impact
closed Payment confirmed & settled Session locked, effects finalized, cashback balance updated Cashback is credited to customer's wallet. Spend counted toward tier progression.
cancelled Payment failed, reversed, or refunded All effects rolled back automatically Cashback is reversed. Spend removed from tier calculation.

Example — Payment Session

Card Transaction: The customer pays with their GCash card at a merchant. GCash sends the merchant details and transaction amount. No cart items are available in this flow — eligibility rules are evaluated on merchant identity, merchant category, and transaction amount alone.

// PUT /v2/customer_sessions/PAYMENT-2026-06-8842 { "customerSession": { "profileId": "GC-USER-98421", "state": "closed", "attributes": { "merchant_id": "MRC-00412", "merchant_name": "Jollibee BGC", "merchant_category": "food_and_beverage", "payment_method": "gcash_card", "channel": "in_store" }, "total": 350.00 } }

POS Transaction: GCash is connected to the merchant's POS system. In addition to merchant details, individual cart items are passed — enabling item-level campaign rules such as cashback on specific product categories or SKUs.

// PUT /v2/customer_sessions/PAYMENT-2026-06-8843 { "customerSession": { "profileId": "GC-USER-98421", "state": "closed", "attributes": { "merchant_id": "MRC-00412", "merchant_name": "Jollibee BGC", "merchant_category": "food_and_beverage", "payment_method": "gcash_qr", "channel": "in_store" }, "cartItems": [ { "name": "Chicken Joy Meal", "sku": "JB-CJ-MEAL-1", "quantity": 2, "price": 139.00, "position": 0, "category": "food_and_beverage" }, { "name": "Peach Mango Pie", "sku": "JB-PMP-1", "quantity": 1, "price": 45.00, "position": 1, "category": "food_and_beverage" } ], "couponCodes": ["GCASH10OFF"], "total": 323.00 } }

Response — Effects for GCash to Render

// Talon.One returns effects to apply { "effects": [ { "effectType": "setDiscount", "props": { "name": "Coupon Discount", "value": 15.95, "position": 0 } }, { "effectType": "addLoyaltyPoints", "props": { "name": "Points Earned", "programId": 12, "value": 320, "subLedgerId": "PH" } } ] }

Loyalty Flow: When a session is closed, Talon.One returns a cashback balance (e.g., "PHP3 earned on this payment"). If the session is cancelled, points are automatically reversed.

🎟️

Active Coupons

All personal coupons allocated to this customer — including status, expiry, and usage count.

🔗

Referral Code

The customer's personal referral code, usage count, and referral campaign details.

🎮

07 — Security

Security

Talon.One is built to meet the security requirements of regulated industries including healthcare distribution. Key controls relevant to GCash's compliance posture are outlined below.

🔒

TLS 1.2+ Everywhere

All API traffic is encrypted in transit using TLS 1.2 or higher. Talon.One does not support unencrypted HTTP connections to any endpoint.

🗝️

API Key Rotation

Integration and Management keys can be rotated at any time without downtime. Gcash's secrets manager should hold keys and rotate them on a 90-day cycle.

👤

Role-Based Access

Talon.One's UI supports granular RBAC — marketing users can manage campaigns without accessing API credentials or financial reporting.

📋

Audit Logs

All Management API calls and UI actions are logged with user identity and timestamp, supporting SOC 2 and internal audit requirements.

🛡️

Data Residency

Talon.One offers regional deployments (EU, US, APAC). For GCash, the APAC region ensures customer data remains within the Asia-Pacific boundary.

✅

SOC 2 Type II

Talon.One holds SOC 2 Type II certification. The latest report is available under NDA for GCash's security review process.
